# Исправление проблемы с видимостью текста на Android

## Проблема
При запуске приложения на Android текст не был виден на белом фоне из-за отсутствия явно заданных цветов для текстовых элементов.

## Решение

### 1. Добавлены недостающие цветовые ресурсы в `Resources/Styles/Colors.xaml`
- Добавлены дополнительные цветовые константы для совместимости:
  - `Primary`, `Secondary`, `Tertiary` - основные цвета приложения
  - `AccentColor` - акцентный цвет
  - `Gray100` - `Gray900` - оттенки серого для различных элементов интерфейса

### 2. Добавлены глобальные стили в `Resources/Styles/Styles.xaml`
- Добавлен глобальный стиль для всех `Label` элементов с явным указанием цвета текста (`PrimaryTextColor` - черный)
- Добавлен глобальный стиль для всех `Button` элементов с белым цветом текста по умолчанию
- Расширены существующие стили для `Entry`, `Editor`, `Picker`, `DatePicker`, `TimePicker` с явными цветами текста

### 3. Исправлены конкретные страницы

#### YandexDiskPage.xaml
- Все `Label` элементы теперь имеют явно заданные цвета текста
- Заменены `{StaticResource Gray600}` на `{DynamicResource SecondaryTextColor}`
- Заменены `{StaticResource Primary}` на соответствующие динамические ресурсы

#### AddEditBookPage.xaml
- Исправлен цвет текста для кнопки "Новый" (было `Grey`, стало `{DynamicResource PrimaryTextColor}`)
- Исправлен цвет текста для кнопки "Отмена" на вторичном фоне

#### UpdateProgressPage.xaml
- Исправлен цвет текста для кнопки "Отмена"

#### BookDetailPage.xaml
- Исправлен цвет текста для кнопки "Редактировать"

## Результат
Теперь весь текст в приложении отображается корректно на Android с хорошей контрастностью:
- Основной текст: черный (`#000000`)
- Вторичный текст: серый (`#666666`)
- Текст на цветных кнопках: белый
- Текст на светлых кнопках: черный

## Применение изменений
Изменения автоматически применятся при следующей сборке приложения. Рекомендуется выполнить полную пересборку проекта для Android:
```
dotnet build -f net9.0-android
```
